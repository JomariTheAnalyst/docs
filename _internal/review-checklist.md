# Documentation review checklist

- [ ] Page has clear `title` and `description` frontmatter.
- [ ] Page follows `AGENTS.md` and `STYLE_GUIDE.md`.
- [ ] Technical behavior has current repository evidence.
- [ ] External settings are verified or clearly marked.
- [ ] Internal links are root-relative and extension-free.
- [ ] Commands and interface labels are exact.
- [ ] Images have files, alt text and redacted content.
- [ ] No password, token, recovery code, private key, customer, order, payment, or environment value appears.
- [ ] Navigation contains only useful published pages.
- [ ] `mint validate` passes.
- [ ] `mint broken-links --check-anchors --check-redirects` passes.
- [ ] `mint a11y` passes.
