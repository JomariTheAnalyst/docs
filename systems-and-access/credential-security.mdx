## What belongs in the password manager

Store passwords, private Storefront tokens, Shopify Admin tokens, Sanity tokens, webhook secrets, Redis tokens, Resend keys, OAuth secrets, recovery codes, and private keys only in the approved password manager.

Use this reference pattern in documentation:

```text
Service: Shopify
Login email: [authorized business email]
Secret location: Password Manager -> SixthGear -> Shopify Production
MFA owner: [verify]
```

## Screenshot rules

Crop screenshots to the task. Remove account avatars, unnecessary email addresses, billing information, customer details, orders, cookies, tokens, recovery material, internal IDs, and browser autofill.

Apply irreversible redaction before export. Review the exported image instead of trusting an overlay in the editor.

## Rotate a credential

<Steps>
  <Step title="Confirm scope and owner">
    Identify every service and environment that uses the credential without copying its value.
  </Step>
  <Step title="Issue a least-privilege replacement">
    Create the replacement in the authoritative platform and store it in the approved password-manager item.
  </Step>
  <Step title="Update approved environments">
    Update only the required local, Preview, or Production scopes. Redeploy when the platform requires it.
  </Step>
  <Step title="Verify and revoke">
    Test the integration, then revoke the old credential. Confirm audit logs show expected activity only.
  </Step>
  <Step title="Record the rotation">
    Record date, owner, affected systems, validation, and password-manager reference. Never record either value.
  </Step>
</Steps>

## Suspected exposure

Restrict the exposed artifact, revoke or rotate the credential, terminate unauthorized sessions, inspect audit logs, update environments, and test the integration. If Git contains the secret, follow an approved history-rewrite incident process; deleting one visible line is not enough.

<Warning>
Never commit secrets to Git. Treat every `NEXT_PUBLIC_*` value as visible to website visitors.
</Warning>
