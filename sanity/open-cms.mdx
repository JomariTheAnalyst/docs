Use this guide after your Sanity invitation has been accepted.

## CMS addresses

<CodeGroup>

```text Local development
http://localhost:7000/studio
```

```text Production (live)
https://www.sixthgearmoto.com/studio
```

</CodeGroup>

Use the local address only when the SixthGear storefront is already running on port `7000`. Use the production address for approved live content work.

## Open the CMS

<Steps>
  <Step title="Choose the correct environment">
    Use local development for developer testing. Use production for approved live content changes.
  </Step>
  <Step title="Sign in with the approved account">
    Use the same account and sign-in method used to accept the invitation. Do not use another employee’s account.
  </Step>
  <Step title="Confirm the workspace">
    Check that the page title shows **Sixthgear CMS** and that the content navigation is visible.

## Common access problems

<AccordionGroup>
  <Accordion title="Access denied">
    Confirm that you signed in with the exact account that received the invitation. If the account is correct, ask the administrator to review your membership and assigned role.
  </Accordion>

  <Accordion title="The wrong Sanity project opens">
    Sign out, then sign in again with the invited account. Ask the administrator to confirm the correct SixthGear project if more than one project is visible.
  </Accordion>

  <Accordion title="The expected content sections are missing">
    Ask the administrator to confirm your role and the content sections currently available in SixthGear CMS. Some sections remain under review.
  </Accordion>

  <Accordion title="The local address does not open">
    Confirm that `npm run dev` is running successfully and that the storefront is available on port `7000`. The local Studio also requires the approved Sanity environment variables.
  </Accordion>

  <Accordion title="The page is blank or keeps loading">
    Refresh once, try a private browser window, and confirm that browser extensions are not blocking the page. Record the browser, time, address, and visible error before asking for support.
  </Accordion>
</AccordionGroup>

<Warning>
  Do not publish a test edit only to confirm access. Open a document and review the available controls instead.
</Warning>
