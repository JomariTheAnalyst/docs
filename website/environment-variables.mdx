Use `.env.example` and `src/lib/env.ts` together. The example lists intended configuration, while the runtime schema shows what current code validates.

<Warning>
Do not add environment values to this page. Store secret values in the approved password manager and configure them in the correct local or Vercel scope.
</Warning>

## Core commerce and infrastructure

| Variable | Purpose | Scope | Secret | Evidence |
| --- | --- | --- | --- | --- |
| `SHOPIFY_STORE_DOMAIN` | Server Shopify API host | Server | No, but operationally sensitive | Required by runtime schema |
| `SHOPIFY_STOREFRONT_ACCESS_TOKEN` | Private Storefront API access | Server | Yes | Required by runtime schema |
| `NEXT_PUBLIC_SHOPIFY_STOREFRONT_TOKEN` | Public Storefront API access | Browser | No; keep scopes limited | Required by runtime schema |
| `NEXT_PUBLIC_SHOPIFY_STORE_DOMAIN` | Browser Shopify API host | Browser | No | Optional schema field; client code uses it |
| `SHOPIFY_API_VERSION` | Storefront API version | Server | No | Defaults to `2025-10` in source |
| `SHOPIFY_ADMIN_ACCESS_TOKEN` | Admin GraphQL access | Server | Yes | Client exists; active caller unconfirmed |
| `SHOPIFY_WEBHOOK_SECRET` | Shopify webhook HMAC check | Server | Yes | Used by `/api/revalidate` |
| `UPSTASH_REDIS_REST_URL` | Redis REST endpoint | Server | Treat as sensitive | Required by runtime schema |
| `UPSTASH_REDIS_REST_TOKEN` | Redis authentication | Server | Yes | Required by runtime schema |

## Vercel and tests

`VERCEL_ENV`, `VERCEL_URL`, and `NEXT_PUBLIC_VERCEL_URL` are platform-provided. `CI` and `TEST_ENV` control Playwright behavior. Verify their scopes in Vercel and GitHub rather than adding local production values.

## Unclear and migration-era names

`REVALIDATION_SECRET`, `SANITY_REVALIDATE_SECRET`, Medusa variables, NextAuth variables, Shopify OAuth identifiers, Payload, Xendit, analytics, Sentry, Klaviyo, and Vercel Blob names remain in source or templates.

Do not remove them until you check both source usage and every external environment scope. Track that work in the internal maintenance register.
